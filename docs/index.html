Health Studio

A local-first personal health dashboard, journal, and tracker.

Overview

Health Studio consolidates mental and physical health management into a single, self-hosted web application deployed via Docker. All data lives locally on the host machine.

Features

  • Journal — Markdown entries with rich editing and preview
  • Metrics — Body metrics tracking with interactive Plotly.js trend charts and 7-day moving averages
  • Results — Exercise results with automatic PR detection across Olympic lifts, powerlifts, CrossFit benchmarks, and running
  • Goals — Goal setting linked to metrics or exercises with progress tracking and Markdown plans
  • Dashboard — Aggregated overview of recent activity, active goals, latest metrics, and PRs
  • Dark/Light mode — Toggle between themes; preference persisted
  • Responsive layout — Collapsible sidebar for mobile
  • CLI — Command-line interface for managing data from the terminal

Architecture

┌────────────────┐      ┌────────────────┐      ┌──────────┐
│   React SPA    │─────▶│  FastAPI        │─────▶│  SQLite  │
│  (Vite + TS)   │ REST │  (Python 3.12) │      │          │
│  port 3000     │      │  port 8000     │      │ data/    │
└────────────────┘      └────────────────┘      └──────────┘
  • Frontend: React 19, TypeScript, Tailwind CSS, Plotly.js, react-markdown
  • Backend: FastAPI, SQLAlchemy, Alembic, py_webauthn
  • Database: SQLite — zero-config, file-based
  • Auth: WebAuthn/Passkeys (passwordless) + scoped API keys

Security

  • Single-user, local-first — no cloud dependencies
  • WebAuthn / Passkey authentication — no passwords
  • Docker containers run as non-root with read-only filesystem
  • Ports bound to 127.0.0.1 only
  • CORS restricted to http://localhost:3000
  • Request body size limited to 1 MB
  • API keys stored as bcrypt hashes
  • No stack traces in production error responses

CLI

Install the CLI for terminal-based access:

cd cli && pip install -e ".[dev]"
hs config init
hs config set url http://localhost:8000
hs config set api-key <your-key>

Available commands: hs dashboard, hs journal, hs metrics, hs results, hs goals, hs config.
